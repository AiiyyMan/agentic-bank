## 5. Light and Dark Mode

### 5.1 Mode Switching Pattern

All semantic tokens resolve differently per mode. The primitive layer is mode-invariant -- the same hex values exist in both modes. Only the semantic mapping changes.

**Pattern:** Light mode uses lower shade numbers for backgrounds (0-10) and higher shade numbers for text (70-90). Dark mode inverts this: higher shades for backgrounds (80-100) and lower shades for text (5-30).

These tokens do NOT change between light and dark modes:

## Appendix: Anomalies and Decisions

1. **indigo.40 = indigo.50** -- Both are #818CF8. Treat indigo.50 as the canonical mid-tone. When building the palette, consider shifting indigo.50 to #6366F1 (Tailwind indigo-500) for better contrast with indigo.60.

2. **pink.5 = pink.10** -- Both are #FDF2F8. Use pink.10 as the canonical light tint. Consider shifting pink.5 to #FEF1FF for differentiation if needed.

3. **Transparent black base** -- Uses #020617 (very dark navy) instead of pure #000000. This creates warmer overlays. Maintain this convention.

4. **Warning text asymmetry** -- Light mode uses `warning.80` (#854D0E) for text but dark mode uses `warning.30` (#FDE047). The light-mode choice of .80 (instead of .70 like other status text tokens) ensures WCAG AA contrast on white backgrounds. The semantic layer in `semantic.json` reflects this intentional asymmetry.

5. **Shadow color** -- All shadows use `rgba(28,25,23,...)` (warm stone-900) rather than pure black. This is mode-invariant -- the same shadow values apply in both light and dark mode per `semantic.json`. For dark mode, consider authoring darker shadow values if needed during implementation.

6. **money.\* typography tokens** -- These are Agentic Bank additions with no SwiftBank equivalent. They use `tabular-nums` for column alignment in transaction lists. Sizes chosen: 36px (balance hero), 24px (transaction row), 16px (inline amounts).

7. **Component token alias resolution** -- Component tokens (e.g., `button.primary.background` = `{color.brand.default}`) reference semantic tokens, which in turn reference primitives. Resolution requires mode context: in light mode, `{color.brand.default}` -> `{color.brand.50}` -> #0EA5E9; in dark mode -> `{color.brand.40}` -> #38BDF8. The implementation in `tokens.ts` pre-resolves these chains per mode.
