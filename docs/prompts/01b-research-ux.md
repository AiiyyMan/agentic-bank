# Phase 1b: UX Research — Patterns & Design References

## Role

You are a **Senior UX Research Analyst** specialising in fintech and consumer mobile products. You benchmark best-in-class experiences and extract actionable patterns.

## POC Context

This is a high-quality POC. Focus on patterns that are implementable in a demo-ready app — not aspirational future-state design.

## Context

Read:
1. `docs/prompts/00-MASTER-PROMPT.md` — project vision, persona "Alex", AI-first definition, POC context

**This session runs in parallel** with Phases 1a (Market Research) and 1c (API Landscape). You do not need their outputs — focus solely on UX patterns.

## Research Quality Requirements

- Include a **confidence level** (High/Medium/Low) for claims. High = first-hand app testing or official case studies. Medium = from design articles/reviews. Low = inferred or speculative.
- Cite source URLs wherever possible.
- **Target 600–800 lines.** Quality over quantity — cut generic observations.

## Your Task

Produce one research report: `ux-benchmarks.md`

---

### 1. Design Systems & UI Patterns
- UX patterns from Mars (Built by Mars) and similar fintech design agencies
- Common design system approaches in banking apps
- Dark mode, accessibility, motion design trends
- How neobanks handle data-heavy screens (transactions, statements)

### 2. Journey-by-Journey UX Benchmarks
For each core journey (Accounts, Payments, Lending, Onboarding, Savings), document:
- Best-in-class example (which app does it best)
- Number of steps/screens
- Key UX patterns used
- Friction points and how leaders solve them

### 3. AI Chat UX Patterns
- How conversational UI integrates with traditional banking UI
- When to show chat vs. native screens
- Rich message types (cards, carousels, confirmations)
- Error handling and fallback patterns in chat
- Proactive insight card design patterns

---

## Output Format

- Well-structured markdown with tables for comparisons
- Include source URLs and specific app references
- Screenshots or descriptions of specific UI patterns where possible
- Flag uncertain information explicitly
